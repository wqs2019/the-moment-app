## 2. 产品背景

### 2.1 问题定义

用户大量时间并不是主动娱乐，而是陷入无意识刷屏：

- 打开手机时并没有明确目的
- 想“放松一下”，最后却刷了 20-40 分钟
- 刷完之后没有获得恢复感，反而更空虚、更焦虑
- 明知应该停下来，却很难马上切换到更有价值的行动

这不是单纯的“自制力问题”，而是用户在当下缺少一个足够低门槛、足够具体、足够即时的替代方案。

### 2.2 市场机会

当前市场上多数相关产品分为两类：

- 强控制类：锁 App、限制时长、番茄钟，容易引发逆反心理
- 泛成长类：习惯打卡、待办清单、冥想内容，离“此刻就能做什么”仍有距离

`别刷啦` 的机会点在于：

- 不直接阻止刷，而是在“想刷的瞬间”提供替代动作
- 用 1-5 分钟的微行动替代 30 分钟的无意识刷屏
- 用“能量损益反馈”让用户看到每次选择的代价和收益

## 3. 产品定位

### 3.1 一句话描述

`别刷啦` 是一个在用户无聊、疲惫、下意识想刷手机时，快速给出 3 个更好选择的轻工具 App。

### 3.2 核心理念

- 不戒手机，只减少无意识刷屏
- 不锁屏幕，而是在每次要刷之前增加一个选择
- 不讲大道理，而是用可执行的小动作帮助用户切换状态
- 不制造焦虑，而是用损益反馈帮助用户自己不想再刷

### 3.3 差异化定位

| 维度 | 别刷啦 | 常见屏幕管理工具 |
| --- | --- | --- |
| 解决方式 | 替代方案优先 | 限制和拦截优先 |
| 用户感受 | 被理解、被提醒 | 被约束、被管理 |
| 行为机制 | 微行动 + 即时反馈 | 时长控制 + 惩罚 |
| 核心闭环 | 推荐任务 -> 选择行为 -> 能量反馈 | 打开 App -> 被阻止 / 记录时长 |

## 4. 产品目标与非目标

### 4.1 MVP 目标

1. 验证“微任务替代刷屏”是否能形成稳定使用闭环
2. 验证“能量损益反馈”是否能提升用户复用意愿
3. 验证“分享型周报/海报”是否具备基础传播能力

### 4.2 阶段性业务目标

- 建立用户首次使用后的 7 日留存基础
- 建立至少一个高频使用场景的产品心智
- 让用户形成“想刷时先打开别刷啦看看”的条件反射

### 4.3 非目标

MVP 阶段暂不追求：

- 完整社交关系链
- AI 大模型个性化推荐
- 复杂会员体系
- 全平台覆盖和企业级管理能力

## 7. 产品核心策略

### 7.1 行为闭环

1. 发现：在想刷或刷了一会儿之后，产品进入用户视野
2. 替代：给出 3 条“此刻可做”的具体任务
3. 执行：用户选择执行任务，或承认自己还是刷了
4. 反馈：立即生成收益或损失反馈
5. 强化：通过账本、周报、海报形成记忆和传播

### 7.2 设计原则

- 极低门槛：不需要准备，不需要切换复杂上下文
- 明确终点：用户知道做多久、做到什么程度算完成
- 具体可执行：不用“自我反思”，而用“发一句消息”“删 10 张图”
- 不说教：不给用户压力，不使用“你应该”“你必须”
- 可视化反馈：让时间损益被看见，而不是抽象存在

## 9. 核心功能详述

### 9.1 无聊解救器

#### 功能描述

用户打开 App 后，系统基于静态画像、动态情境和行为反馈，推荐 3 条此刻可做的任务。

#### 推荐任务要求

- 1-5 分钟可完成
- 不需要额外准备
- 有具体动作和明确终点
- 可匹配当前场景与情绪
- 具有可感知的轻收益

#### 首页卡片展示字段

- 任务标题
- 预计时长
- 预期能量值
- 任务类型标签
- 开始按钮

#### 核心流程

1. 用户进入首页
2. 首页展示 3 条推荐任务
3. 用户点开任务详情
4. 用户点击“开始”
5. 用户完成后点击“完成了”
6. 系统记录收益并展示即时反馈
7. 若用户放弃，点击“算了，我还是去刷了”
8. 系统记录损失并进入损益反馈页

#### 异常与兜底

- 无网络时展示本地通用任务
- 推荐失败时展示 3 条通用任务
- 用户短时间多次刷新时限制刷新频率，避免“任务焦虑”

### 9.2 能量账本

#### 功能描述

记录每次“做了更好的事”与“还是去刷了”的选择结果，并用能量值进行即时反馈和长期统计。

#### 记录内容

| 行为类型 | 字段 |
| --- | --- |
| 完成任务 | 任务名、时长、获得能量值、完成时间 |
| 去刷了 | 刷屏时长、内容类型、消耗能量值、记录时间 |

#### 反馈方式

- 即时反馈：本次收益 / 本次损失
- 差值反馈：本可以获得多少，实际损失多少
- 周报反馈：本周净收益、损失高峰、最优场景

#### 示例文案

- 完成反馈：你刚刚完成了“给朋友发一句问候”，获得 `+18` 能量。
- 损失反馈：你刷了 `35` 分钟短视频，消耗了 `42` 能量。这段时间原本可以完成 3 个小任务。

### 9.3 智能推荐引擎

#### 目标

让任务推荐越来越懂用户，而不是每次随机给任务。

#### 三维数据模型

| 层级 | 数据来源 | 采集方式 | 作用 |
| --- | --- | --- | --- |
| 静态画像 | 身份、重点场景 | Onboarding 主动填写 | 冷启动过滤不相关任务 |
| 动态情境 | 时间、天气、工作日、位置类型 | 系统自动获取 | 让推荐适合“此刻” |
| 行为反馈 | 点击、完成、放弃、评分、刷屏记录 | 自动记录 | 优化推荐准确率 |

#### MVP 实现方案

- 规则引擎 + 标签系统
- 20-30 条规则覆盖主要场景
- 暂不引入重模型推荐

#### 冷启动方案

新用户前 7 天视为数据采集期，用选择题快速建立偏好：

- 你现在更想：
  - 有成就感
  - 平静下来
  - 连接他人
  - 找点乐子

### 9.4 提醒与触发机制

#### 目标

在用户还没彻底刷进去之前，让产品被看见。

#### 三层触发策略

| 层级 | 方式 | 触发条件 | 说明 |
| --- | --- | --- | --- |
| 视觉锚点 | 桌面小组件 | 每次解锁手机可见 | 展示“此刻可做的 1 件小事” |
| 视觉锚点 | 锁屏壁纸生成器 | 用户主动设置 | 让“别刷啦”成为环境提醒 |
| 刷后触发 | 推送通知 | 连续刷屏 10-15 分钟后 | 用轻提醒打断自动化行为 |
| 物理中断 | 无聊仪式 | 用户主动设置 | 屏幕朝下、深呼吸、再决定刷不刷 |

#### MVP 优先实现

- 桌面小组件
- 刷后 10 分钟提醒
- 锁屏壁纸生成器

## 11. 任务库内容体系

### 11.1 标签结构

| 标签维度 | 可选值 |
| --- | --- |
| 适合人群 | 大学生 / 职场新人 / 资深打工人 / 年轻父母 / 通用 |
| 典型场景 | 睡前 / 通勤 / 工作间隙 / 独处 / 排队等待 / 通用 |
| 任务类型 | 轻学习 / 创作表达 / 社交连接 / 治愈放松 / 整理收纳 / 感官体验 |
| 预估时长 | 1-2 分钟 / 3-5 分钟 |

| 氛围标签 | 安静 / 活跃 / 怀旧 / 新鲜感 / 治愈 / 成就感 |

### 11.2 内容原则

- 任务必须以动词开头：拍 / 发 / 删 / 写 / 听 / 关 / 数 / 翻
- 任务必须有明确终点
- 任务必须低准备成本
- 禁止空泛说教和抽象鸡汤

| 任务完成率 | 任务设计是否合理 |
| 放弃后记录率 | 用户是否愿意面对损失反馈 |
| 小组件添加率 | 触发机制是否成立 |

## 19. 风险与假设

### 19.1 核心假设

- 用户愿意在“想刷”时多看一眼替代方案
- 足够具体的小任务能显著提高执行率
- 损益反馈不会让用户感到被羞辱，反而会增强自我觉察
- 周报和海报具备基础分享动机

### 19.2 主要风险

| 风险 | 说明 | 应对方案 |
| --- | --- | --- |
| 推荐不准 | 用户会快速流失 | 用规则引擎快速迭代，优先优化高频场景 |
| 任务太空泛 | 用户不会执行 | 强化任务模板与终点定义 |
| 提醒过多 | 用户关闭通知 | 严控频率，优先做轻提醒 |
| 反馈太沉重 | 用户产生挫败感 | 控制文案语气，强调选择而非责备 |

## 21. 附录

### 21.1 成功标准

如果用户在第一周内至少完成以下任意两项，则可以视为核心价值被验证：

- 完成 3 次以上替代任务
- 主动查看能量账本 2 次以上
- 添加桌面小组件
- 生成并分享周报或海报 1 次

### 21.2 产品判断标准

做任何新功能时，优先问三个问题：

1. 它是否能帮助用户在“想刷”的瞬间更容易做出别的选择？
2. 它是否比现有体验更轻、更具体、更不让人焦虑？
3. 它是否能增强“推荐 -> 行动 -> 反馈 -> 复用”的闭环？
